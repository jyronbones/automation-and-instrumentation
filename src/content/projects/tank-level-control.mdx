## Problem description

The task is to keep a tank's level inside a band, run a pump to do it, and raise
an alarm before it overflows.

A single setpoint makes the pump **short-cycle** at the threshold, switching on
and off rapidly and wearing out the motor, contactor, and pump. The fix is a
**deadband (hysteresis)** between two setpoints. On top of the automatic control,
the project adds an operator **Hand-Off-Auto** selector and a **fail-safe overflow
alarm**.

This is a *fill* tank: a pump draws from a source and holds the level between a
low and a high float.

## I/O assignment

| Tag | Device | Type | Address |
| --- | --- | --- | --- |
| `LSL` | Low-level float | NO (true at the low setpoint) | `%IX0.0` |
| `LSH` | High-level float | NO (true at the high setpoint) | `%IX0.1` |
| `LSHH` | High-high (overflow) float | NO (true at overflow) | `%IX0.2` |
| `HAND` | HOA selector — Hand | NO | `%IX0.3` |
| `AUTO` | HOA selector — Auto | NO | `%IX0.4` |
| `RESET` | Acknowledge / reset pushbutton | NO, momentary | `%IX0.5` |
| `PUMP` | Pump contactor coil | Output | `%QX0.0` |
| `ALARM` | Overflow horn / beacon | Output | `%QX0.1` |

> `PUMP_CALL` — the fill latch — is an internal memory bit, not wired I/O.
> Discrete float switches keep the logic readable; the same scheme scales to an
> analog level transmitter (`LT`, 4–20 mA) by comparing the measured level to
> numeric setpoints with a deadband — the control concept is identical.

## Logic explanation

### Automatic mode — two-point control with deadband

The core is a **set/reset latch**, `PUMP_CALL`, that keeps the pump running for a
full fill:

```
SET   PUMP_CALL   when the low float makes   (LSL = 1, level at the low setpoint)
RESET PUMP_CALL   when the high float makes  (LSH = 1, level at the high setpoint)
```

Set at the low float and reset only at the high float, the pump runs the tank
from low all the way up to high, then stays off until the level falls back to
low. The gap between `LSL` and `LSH` is the **deadband** — it's what eliminates
short-cycling. In auto:

```
Pump (Auto) = AUTO AND PUMP_CALL AND NOT ALARM
```

### Hand mode — manual override, interlock retained

The **Hand-Off-Auto** selector is standard plant practice: *Hand* for
maintenance/commissioning, *Off* to lock out, *Auto* for normal running. In Hand
the pump runs continuously regardless of level **except** that the overflow
interlock is still enforced — a manual mode should never be able to flood the
tank:

```
Pump (Hand) = HAND AND NOT ALARM
Pump = Pump(Auto) OR Pump(Hand)
```

With the selector in **Off**, neither path is true, so the pump is positively
stopped.

### Overflow protection — latched, fail-safe

If the level ever reaches the high-high float `LSHH`, something has failed (a
stuck `LSH`, a control fault, a stuck pump). The response is latched so the event
can't be missed:

- An `ALARM` latch sets on `LSHH`.
- `ALARM` forces the pump **off** in every mode and energizes the horn.
- The latch holds until the level has recovered **and** the operator presses
  `RESET` — so the alarm can't simply clear itself the moment the level dips.

### The logic (three rungs)

The finished OpenPLC program is three rungs: the fill latch, the overflow-alarm
latch, and the pump output.

**Rung 1 — auto fill latch.** `LSL` in parallel with the `PUMP_CALL` seal
contact, in series with a normally-closed `LSH`. Starts on low level, seals in,
and drops out at high level — the two-point hysteresis:

```
  LSL              LSH        PUMP_CALL
|--] [------+------]/[---------(  )-------|
|           |                             |
| PUMP_CALL |                             |
|--] [------+                             |
```

**Rung 2 — overflow alarm latch.** Same seal-in pattern, tripped by `LSHH` and
cleared only by a normally-closed `RESET`:

```
  LSHH             RESET      ALARM
|--] [------+------]/[---------(  )-------|
|           |                             |
|  ALARM    |                             |
|--] [------+                             |
```

**Rung 3 — pump output.** `HAND` in parallel with (`AUTO` **and** `PUMP_CALL`),
then a normally-closed `ALARM` in series. The pump runs on a manual Hand call or
on an automatic fill call — but the `ALARM` interlock opens the rung and forces
it off regardless of selector position:

```
  HAND                    ALARM       PUMP
|--] [-----------+---------]/[---------(  )-------|
|                |                                |
| AUTO  PUMP_CALL|                                |
|--] [---] [-----+                                |
```

In boolean terms:

```
PUMP_CALL = (LSL OR PUMP_CALL) AND NOT LSH          ' fill latch (hysteresis)
ALARM     = (LSHH OR ALARM) AND NOT RESET           ' overflow latch
PUMP      = (HAND OR (AUTO AND PUMP_CALL)) AND NOT ALARM
```

### Sequence of operation

| Condition | Pump | Alarm |
| --- | --- | --- |
| Auto, level above `LSL` | Off (idle) | — |
| Auto, level falls to `LSL` | On (fills to `LSH`) | — |
| Auto, level reaches `LSH` | Off | — |
| Hand selected | On | — |
| Off selected | Off | — |
| Level reaches `LSHH` | Forced Off | On (latched) |
| `RESET` after recovery | resumes mode | Clears |
