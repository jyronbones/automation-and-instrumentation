## Problem description

A momentary **Start** button energizes the motor, a momentary **Stop** button
de-energizes it, and the motor keeps running after the operator releases Start. On
top of that, an **emergency stop** and the motor's **thermal overload** must be
able to stop it at any time.

This is the standard **3-wire control circuit**, implemented in ladder logic and
verified in the OpenPLC simulator.

## I/O assignment

| Signal | Device | Type | Address |
| --- | --- | --- | --- |
| Start | Pushbutton | NO, momentary | `%IX0.0` |
| Stop | Pushbutton | NC, momentary | `%IX0.1` |
| E-Stop | Mushroom head | NC, maintained | `%IX0.2` |
| Overload (OL) | Thermal overload aux | NC, maintained | `%IX0.3` |
| Reset | Pushbutton | NO, momentary | `%IX0.4` |
| Motor | Contactor coil | Output | `%QX0.0` |
| Run light | Pilot light | Output | `%QX0.1` |
| Fault light | Pilot light | Output | `%QX0.2` |

## Logic explanation

The heart of the circuit is the **seal-in (latch)**. A normally-open auxiliary
contact of the motor output is wired in parallel with the Start button. The
control behaves like this:

- **Start (NO, momentary).** Pressing Start completes the rung and energizes the
  motor coil — but only for as long as the button is held, on its own.
- **Seal-in contact.** The instant the motor energizes, its parallel NO contact
  closes and keeps the rung true after Start is released. The output "seals
  itself in." This is why a momentary button gives continuous run.
- **Stop (NC, momentary).** Wired in series. Pressing it breaks the rung, the
  coil drops out, the seal-in contact opens, and the motor stays off until Start
  is pressed again.

The safety interlocks sit ahead of the start/stop logic, wired **normally closed**
so the circuit is **fail-safe** — a broken wire, a lost connection, or a tripped
device all read as "stop," never "run":

- **E-Stop (NC).** Operating it (or any wiring fault) drops the coil immediately
  and overrides the seal-in. It isn't latched — once released, the line is ready
  to start again.
- **Stop (NC).** Breaks the rung for a normal controlled stop.

### Overload fault latching

The overload gets one extra layer. Instead of de-energizing only while the trip is
present, it **sets a latched fault**:

- When the thermal overload trips, a **Fault** memory bit latches on.
- The fault holds the motor down *and* lights a dedicated **Fault** indicator.
- It does **not** clear on its own when the overcurrent goes away — the motor will
  not silently restart. The operator must clear the cause and press **Reset**, and
  Reset only takes effect once the overload has actually cleared.

This "trip → latch → require manual reset" pattern is standard industrial practice:
a fault should demand a human acknowledgement, not vanish on its own.

A **Run** indicator mirrors the motor coil and a **Fault** indicator mirrors the
fault bit — a simplified operator HMI.

### The logic (five rungs)

The finished OpenPLC program is five rungs: control, run indication, fault memory,
reset, and fault indication.

**Rung 1 — motor seal-in.** Four normally-closed interlocks (E-Stop, Fault,
Overload, Stop) in series, then Start in parallel with the Motor seal contact:

```
  E-Stop Fault  OL   Stop       Start         Motor
|--]/[---]/[---]/[----]/[----+---] [---+--------( )------|
|                            |         |                 |
|                            |  Motor  |                 |
|                            +---] [---+                 |
```

**Rung 2 — run light.** Mirrors the motor output:

```
  Motor                            Run light
|--] [-----------------------------( )-----------------|
```

**Rung 3 — fault latch.** An overload sets the Fault bit, which seals itself in:

```
  Overload                         Fault
|--] [------+----------------------( )-----------------|
|           |                                          |
|  Fault    |                                          |
|--] [------+                                          |
```

**Rung 4 — fault reset.** A reset coil: pressing Reset clears the Fault bit. If
the overload is still present, Rung 3 simply re-latches it on the next scan:

```
  Reset                            Fault
|--] [-----------------------------(R)-----------------|
```

**Rung 5 — fault light.** Mirrors the Fault bit:

```
  Fault                            Fault light
|--] [-----------------------------( )-----------------|
```

In boolean terms:

```
Motor      = (Start OR Motor) AND NOT E-Stop AND NOT Fault AND NOT Overload AND NOT Stop
Fault      = Overload OR Fault                  ' latched (seal-in)
Fault      -> reset (R) on Reset                ' re-latches via R3 if still overloaded
RunLight   = Motor
FaultLight = Fault
```

The Overload appears twice on purpose: directly in Rung 1 for an immediate stop,
and in Rung 3 to latch the fault so the motor can't restart until it's cleared and
Reset is pressed. Rung 4's `(R)` just resets the Fault bit — and because Rung 3
seals itself in, the fault re-latches on the next scan if the overcurrent is still
present, so Reset only takes hold once the overload has actually cleared.

### Sequence of operation

| Action | Result |
| --- | --- |
| Press Start | Motor energizes and seals in; Run light on |
| Release Start | Motor keeps running (seal-in holds) |
| Press Stop | Motor stops; ready to restart |
| Press E-Stop | Motor stops immediately; ready once released |
| Overload trips | Motor stops; Fault latches; Fault light on |
| Press Reset (overload cleared) | Fault clears; ready to restart |
| Press Reset (still overloaded) | Fault re-latches; clears only once OL is gone |

A real benefit of doing this in a PLC rather than purely with relays: the seal-in
is logical, not a physical contact that can weld or chatter, and the same inputs
can drive interlocks, run-hour counters, and alarms without rewiring.

## Live simulation

A JavaScript twin of the ladder logic above, running on a ~50 ms scan loop with
the same **read inputs → solve logic → write outputs** cycle a real PLC uses. Hold
**Start** and release it: the seal-in keeps the motor running. **Stop** and
**E-Stop** drop the coil. Trip the **Overload** to latch a fault and light the
**Fault** lamp, then clear it and press **Reset** to restart. (A faithful
